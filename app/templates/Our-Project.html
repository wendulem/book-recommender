Book Recommender

Our Product Overview

The Book Recommender makes it quick and easy to find the next great book to read. With the help of AI and our team, our tool is able to effectively predict what you would want to read based on 3 of your favorite books. 

learn more about our team
Easy to Use

Start typing the name of your book and click on the corresponding pop-up.

Effective

Our tool will effectively select similar books based on book summaries and allow you to pick and choose what to read next.

Simple

When you've finished putting in 3 of your favorite books, click 'Submit' and our tool will start the book recommending process.

Informative

Unsure of whether or not you'll like the recommendations? Click on each title for a pop-up with a brief summary of the book.

Our Mission

It can be hard to find the right book. With all the genres, subgenres, and writing styles, it can be overhwleming when you finish a great book and want to read something just as good if not better than what you just read. A simple google search won't do. That's where our Book Recommender comes in. We've created a tool that not only makes it easy to find your next read, but for people who want to explore similar books for any purpose.

The Process

  • Data Collection
  • Data Cleaning
  • Model Training
  • Website

After we determined our mission, we browsed through Kaggle to find a dataset. Kaggle allows users to find and publish data sets, explore and build models in a web-based data-science environment, work with other data scientists and machine learning engineers, and enter competitions to solve data science challenges. Then after voting on which data to use and how to use it, we collectively decided on the book summary data. It provides information on the book’s Wikipedia ID, free database id, title, production date, genre, and plot summary. We then downloaded the file and imported it into CoCalc to further analyze the data.

After we collected our dataset, we realized that the book titles and descriptions were already in the proper format, so we had very little cleaning to do. This meant that all we needed to do was filter the data by removing blank entries, duplicates, and unnecessary data such as the publisher or book ID. This allowed our dataset to be in the proper form for training using doc2vec. 

To train our model, we used Gensim’s doc2vec algorithm. Gensim is an open source python library for representing the semantic meaning of words, through methods like word2vec, which can represent the relative meanings of words through vectors. Doc2vec is an adaptation of word2vec, which combines these word vectors to produce a vector for each document in the dataset. Given a book in the dataset, we can then use the vector model to find the most similar books to it. However, searching through every possible document to find the exact most similar vector is inefficient and unnecessary, so we used Spotify’s Annoy indexer to find approximate results in a much more time-efficient manner. 

For our website, we used the app NicePage to customize and create the design of our website and imported it into our server. There were many issues surrounding the inability for CSS to show up when we pull the website up, and the files themselves. After exporting the HTML files from NicePage with the CSS successfully, we were able to connect the HTML files to flask, which is a python module that assists the web development process. After that, we debugged the integration of our book recommending function into our website.
